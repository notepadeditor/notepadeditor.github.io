Cookies & Storage Policy

Last Updated: June 22, 2026

This policy details how Notepad Editor Online (notepadeditor.github.io) uses tracking components, programmatic session items, and local browser memory to ensure a stable writing experience.

1. Does This Site Use Cookies?

No, this application does not deploy typical HTTP tracking cookies to record your identity across third-party networks. Instead, we use your browser's internal Web Storage API (Local Storage) to handle configuration workflows and manage draft recovery states seamlessly.

2. How We Use Local Storage

Local Storage allows us to keep key editor settings persistent directly on your hardware so you don't lose progress between sessions. We use this mechanism for the following features:

  • Document Draft Protection: Your raw note text is auto-saved locally every few seconds so that if you refresh the browser or close a tab, your work is restored instantly when you return.
  • Editor Personalization: Saves custom application flags such as your chosen font size step, font formatting layouts, line-wrap switches, and Dark Mode settings.

3. Third-Party Web Storage Usage

We do not share local storage properties with data monitoring brokers or digital advertising hubs. If infrastructure components or static web font CDNs require tracking states to route data smoothly, those layers operate completely outside our system cache boundaries.

4. How to Manage and Delete Your Data

Because local storage configurations reside completely inside your own user terminal, you can erase this text memory loop whenever you choose. To reset your editor, you can either:

  • Click the Trash/Clear Editor (🗑) button inside our interface to wipe current working buffers instantly.
  • Clear your browser's application cache, site data, or localized offline structural cookies history using your browser's built-in privacy settings panel.
